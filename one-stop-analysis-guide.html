Overview

One-Stop Analysis is a browser-based tool for merging plate-based experimental data with metadata, computing statistics, and exporting Prism-ready tables. No installation required — everything runs locally in your browser.

All computation happens client-side. No data is sent to any server.

Step 1 — Upload Files

Metadata File

A CSV or TSV that maps each well to experimental conditions. Typically contains columns like Well, Plate, Treatment, Dose, siRNA, etc.

The metadata file can contain any number of annotation columns — you'll choose which ones to include in Step 2.

Result File(s)

One or more CSV/TSV files containing measurement values (e.g. fluorescence, luminescence, absorbance). Multiple files are merged column-by-column using the Well (and optionally Plate) key. Each file can have different measurement columns.

Merge Key

The merge key defines which columns to use for joining the metadata and result tables.

Well column
The column that contains the well identifier (e.g. A01, A1, B03). Well formats are normalized automatically — A1 and A01 are treated as the same well.
Required in both metadata and every result file. The tool will warn if well coverage is <100%.
Plate column
An optional column identifying the plate number or name (e.g. Plate1, P1). When set, merging uses Plate + Well as the composite key, allowing multiple plates per file.
Use when your experiment spans multiple plates within a single file. Leave blank for single-plate files.
If Metadata has a Plate column but a Result file does not, the merge falls back to Well-only for that file, and a warning is shown. The Plate column on the Metadata side must be cleared if you want Well-only matching across all files.

Step 2 — Configure Columns

Metadata Columns

Select which metadata annotation columns to carry through into the merged dataset. Deselect any columns that are irrelevant to your analysis (e.g. internal IDs, barcodes).

Click Select All to include all columns. The output alias field lets you rename a column in the merged output.

Columns to Merge

For each result file, select which measurement columns to include. You can rename each column with a custom alias.

Filter Rows / Wells

Omit column + Omit value
Mask (hide from analysis) all rows where the specified column equals the specified value. The default is to omit rows where a QC or Flag column equals 1.
Use to exclude failed wells, edge wells, or QC-flagged measurements. Masked wells are shown in grey in the merged table and excluded from all statistics.
Omit wells
Manually specify wells to mask, e.g. A1, B03, H12. Both zero-padded (A01) and unpadded (A1) formats are accepted.
Use for known bad wells (e.g. stuck dispenser, bubble) that don't have a QC flag column.
Include omit column
When checked, the "Omit column" is still exported in the merged CSV even though it's used for masking.
Remove all-empty columns
After masking, any column where every non-masked value is empty is dropped from the output.

Computed Columns

Add derived columns calculated from existing result columns using a formula. First map result columns to single-letter variables (a, b, c…), then write a formula like (a-b)/c or a/b*100.

Formulas support standard arithmetic (+, −, ×, ÷), power (**), and parentheses. The result is computed row-by-row.
If a formula references a column with missing values, the computed value for that row will be NaN and excluded from statistics.

Step 3 — Results

Analysis Columns (Apply to all)

Plate column
If set, all statistics are computed independently per plate. Heatmap and bar chart switch to per-plate view.
Value column
The measurement column used for all statistics (heatmap, bar chart, stats tables).
Group by
One or more metadata columns used to define experimental groups (e.g. Treatment, or both siRNA and Dose). Groups are the combination of all selected columns.

Plate Heatmap

A spatial visualization of measurement values in plate-well layout. Color gradient ranges from the minimum (cool) to maximum (warm) value on the plate. Hover over any cell to see the exact value and well ID.

Use Small / Large to adjust cell size. Export Figure saves as PNG.

Bar Chart

Mean ± SEM per group. Used for quick visual QC of group separation before running formal statistics.

Statistics Table — Assay Window

Compares each experimental group against a designated control group within each plate. Reports a comprehensive set of effect size and quality metrics plus Welch's t-test p-value.

Settings

Group by
Columns defining how wells are assigned to groups for this tab (can differ from the global Group by).
Plate column
If set, statistics are computed per plate separately. Leave blank to pool all plates.
Control group
The reference group all other groups are compared to (e.g. DMSO, NTC, WT).
Select the negative control — the baseline condition with no expected effect.
Outlier removal
Method to detect and exclude outlier values within each group before computing statistics. See Outlier Removal section for details.
Use Grubbs for small n (3–6). Use ROUT as a more robust alternative. Use IQR for larger n or non-normal data. Leave as none if you prefer to flag outliers manually.
Correction (p)
Multiple testing correction applied to the Welch t-test p-values. None: raw p-values. Holm (FWER): controls family-wise error rate. BH (FDR): controls false discovery rate. Correction is applied per plate (within-plate family).
Use Holm when you need high confidence that each significant result is real (confirmatory). Use BH for exploratory screening with many groups where some false positives are acceptable.

Statistics Table — Disease Window

Extends the Assay Window by adding a Disease column dimension. Compares "disease" vs "healthy" (or any two disease states) within each experimental group combination.

Example: Groups = siRNA + Dose. Disease column = Condition with values Normal / Disease. The table shows how each siRNA × Dose combination differs between Normal and Disease conditions.

Disease column
The metadata column encoding disease state (e.g. Condition, Cell_type). This column becomes the comparison axis rather than a grouping factor.
Use when your plate has both healthy and disease-model wells, and you want to compute effect sizes for the "disease effect" within each treatment group.
Disease control
The reference value within the Disease column (e.g. Normal). All other disease values are compared to this.

Statistics Table — Dose Group

Runs dose-response analysis (EC50 fitting) and 2-way ANOVA on grouped dose-response data.

EC50 checkbox
Fits a 4-parameter logistic (4PL) curve to each group's dose-response data and extracts the EC50. See EC50 / 4PL.
Enable when your experiment has a dose-response design (multiple concentrations per compound/treatment).
2-way ANOVA checkbox
Runs a 2-way ANOVA (Group × Dose) using Type III sum of squares, followed by pairwise post-hoc comparisons with multiple testing correction.
Enable to test whether different treatment groups respond differently across doses (interaction effect).
Dose column
The column containing dose/concentration values (numeric). Used as the X-axis for EC50 fitting and ANOVA.
0 in log as
Dose = 0 cannot be represented on a log scale. This setting replaces 0 with the chosen small value (0.001, 0.01, 0.1) or excludes those wells entirely (exclude).
Use 0.01 (default) to keep vehicle wells on the plot. Use exclude if the vehicle wells would distort the curve fit.
Comparison
vs Reference: each group is compared only to the reference group (Dunnett-style). All pairs: all possible group pairs are compared.
Use vs Reference when you have a clear negative control group and want to test each treatment against it. Use All pairs when no single reference exists and all groups are equally interesting.
Correction
Holm (FWER): controls probability of any false positive. BH (FDR): controls proportion of false positives among significant results.

Step 4 — Prism Export

Reshapes the merged data into GraphPad Prism table format (TSV). Click 📋 Copy to Prism then paste directly into a Prism table. A preview chart is rendered in-browser with the same layout as Prism.

Common Settings

Figure output by
Split the output into separate tables/figures by a metadata column (e.g. Plate, Assay). Each unique value of that column becomes one Prism table.
Use when you want one Prism file per plate, or one figure per compound.
Outlier (Prism)
Removes outliers from individual replicates before computing mean/SEM for the Prism table. Uses the same methods as the Statistics Table (Grubbs, IQR, ROUT).
Group by
The metadata columns that define groups (columns in the Prism table). Each unique combination becomes one data set (column group) in Prism.
Filter col + ≠ value
Exclude rows where the specified column equals the specified value before building the Prism table. Useful for removing vehicle wells from a dose-response table.
Separator
Character used to join multiple Group by column values into a single label (e.g. siRNA_Dose with separator _).

XY Chart

Dose-response format. X axis = dose/concentration. Y axis = mean ± SEM. Each group becomes one curve. Output matches Prism XY table.

X axis
The column used as the X axis — typically your dose/concentration column.
X sort
Numeric: sort dose values numerically (0.001, 0.01, 0.1…). Alpha: sort alphabetically. As-is: preserve original order in the data.
Use Numeric for concentration series. Use As-is if your data is already in the right order.
Log₁₀ X axis
Display the X axis on a log₁₀ scale. Standard for dose-response curves (concentrations span multiple orders of magnitude).
4PL fit preview
Overlays a fitted 4-parameter logistic curve on each group in the preview chart. Does not affect the exported Prism table — Prism performs its own fitting.
2-way ANOVA
Runs 2-way ANOVA (Group × Dose, Type III SS) on the individual data points and annotates the preview chart with significance stars at each dose point.
EC50
Fits 4PL to each group and displays EC50 values in the legend of the preview chart.
0 in log as
Visible only when Log₁₀ is active. Replaces dose = 0 with a small proxy value so it can appear on the log axis. Same options as in the Dose Group tab.
Control group (ANOVA)
The reference group for ANOVA post-hoc comparisons. Visible only when 2-way ANOVA is checked.
Correction (ANOVA)
Multiple testing correction for ANOVA post-hoc p-values. Holm (FWER) or BH (FDR).
MSE from
All groups (Prism): uses the pooled mean squared error from all groups in the ANOVA model (same as Prism/emmeans). Gives more statistical power when groups share similar variance. Per-pair only: recomputes MSE using only the two groups being compared, relaxing the equal-variance assumption.
Use All groups to match Prism's default behavior. Use Per-pair only if groups have very different variances or if you want a more conservative test.

Column Chart

Bar plot format. Each group = one bar (mean ± SEM). Rows in the Prism table = replicates. Output matches Prism Column table.

Row factor
An optional metadata column used to create sub-rows within each group (e.g. replicate ID, biological repeat). When set, each unique value becomes a separate row in the Prism table.
Row sort
How to order the rows/groups in the output: Numeric, Alpha, or As-is.
Welch's t-test
After checking this box, significance stars are drawn on the preview chart using Welch's t-test for each selected comparison pair.
Comparison (t-test)
vs Control: each group is compared only to the designated control group. All pairs: all possible group pairs are compared.
Use vs Control for 30 siRNAs vs NTC. Use All pairs when every group comparison is of interest.
Control group (t-test)
The reference group when Comparison = vs Control. Populated automatically from the current groups.
Correction (t-test)
None: raw p-values, no adjustment. Holm (FWER): recommended when comparing many groups to one control and each star needs to be reliable. BH (FDR): allows a controlled proportion of false positives.
Use None for 2–3 comparisons. Use Holm for 10+ groups vs control. Use BH for high-throughput screening where you plan to validate hits anyway.
Star style
Top of bar: stars appear above each significant test group bar, colored to match the bar. Bracket: classical bracket-style annotation connecting the two compared groups, sorted by span to minimize overlap.
Use Top of bar for vs-control comparisons (clean, uncluttered). Use Bracket for all-pairs comparisons where relationships between groups matter.

Grouped Chart

Grouped bar plot. Rows = one grouping factor (e.g. treatment, siRNA). Columns = another grouping factor (e.g. cell line, condition). Output matches Prism Grouped table.

Row factor
The metadata column that defines the rows (X-axis clusters) in the Prism table (e.g. Treatment).
Group by (columns)
The metadata column(s) that define the bar groups within each cluster (e.g. Cell_line). Each unique combination becomes one bar per cluster.
Welch's t-test (Grouped)
Same as Column chart, but comparisons are performed between column groups within each row cluster. For example, if comparing WT vs KO for each siRNA, each siRNA cluster gets its own independent t-test.
Use to annotate which row clusters show a significant difference between column groups (e.g. which siRNAs show a significant WT vs KO difference).
Comparison (Grouped t-test)
vs Control col: each column group is compared to the designated control column within each row cluster. All col pairs: all column-group pairs within each cluster.
Correction (Grouped t-test)
Family = all row × pair tests across the entire figure (e.g. 5 rows × 3 column pairs = 15 tests corrected together).

Statistical Terms Glossary

Descriptive Statistics

TermFormula / DefinitionInterpretation
Mean (x̄) Σxᵢ / n Average value. Sensitive to outliers.
SD √[Σ(xᵢ−x̄)² / (n−1)] Spread of individual values around the mean. Larger SD = more variable group.
SEM SD / √n Uncertainty in the estimated mean. Used as error bars in bar charts. Decreases as n increases.
CV% SD / |mean| × 100% Relative variability. CV <10% is typically excellent for HTS assays; <20% is acceptable.
Median Middle value when sorted Robust alternative to mean. Less affected by outliers or skewed distributions.
MAD 1.4826 × median(|xᵢ − median|) Median Absolute Deviation, scaled by 1.4826 (= 1 / Φ⁻¹(0.75)) so that MAD ≈ SD for normally distributed data. This factor makes MAD a consistent estimator of σ, allowing direct comparison with SD-based metrics. Without scaling, MAD ≈ 0.674 × SD for a normal distribution.

Effect Size Metrics

Fold Change (FC)

FC_mean = mean_exp / mean_ctrl
FC_median = median_exp / median_ctrlRobust

How many times larger (or smaller) the experimental value is relative to control. FC = 2 means the signal doubled; FC = 0.5 means halved; FC = 1 means no change. The median-based version is more robust to outlier wells distorting the group average.

FC is undefined when the control mean/median is 0. Fold change is scale-dependent — FC of 2 is more meaningful when both values are well above the detection limit.

Z-score

Z_mean = (mean_exp − mean_ctrl) / SD_ctrl
Z_median = (median_exp − median_ctrl) / MAD_ctrlRobust

How many spread-units the experimental location is above or below the control. Used in HTS to rank hits; |Z| > 2 is a common preliminary threshold.

The median version substitutes MAD (scaled by 1.4826) for SD_ctrl. Because MAD is insensitive to extreme values, Z_median is more stable when the control group has a few outlier wells — a common occurrence in HTS plates.

SSMD (Strictly Standardized Mean Difference)

SSMD_mean = (mean_exp − mean_ctrl) / √(SD_exp² + SD_ctrl²)
SSMD_median = (median_exp − median_ctrl) / √(MAD_exp² + MAD_ctrl²)Robust

Similar to Z-score but accounts for variability in both groups simultaneously. Widely used in RNAi and compound screening QC. The median version uses MAD (×1.4826) in place of SD, providing the same robustness advantage as Z_median — particularly useful when both the experimental and control groups may contain outliers.

|SSMD|Hit classification
>3Extremely strong
2–3Strong
1–2Moderate
<1Weak / no effect

Z' Factor

Z'_mean = 1 − 3(SD_exp + SD_ctrl) / |mean_exp − mean_ctrl|
Z'_median = 1 − 3(MAD_exp + MAD_ctrl) / |median_exp − median_ctrl|Robust

A plate quality metric quantifying the separation between positive (PC) and negative (NC) control populations. The factor 3 corresponds to 3σ coverage (~99.7% of a normal distribution). Z' ≥ 0.5 means the PC and NC distributions are well separated with no overlap at the 3σ level.

The median version substitutes MAD (×1.4826) for SD. On HTS plates where a few stuck dispensers or edge effects create outlier control wells, Z'_median gives a more reliable read of true assay quality than Z'_mean.

Z'Plate quality
≥ 0.5Excellent — suitable for HTS
0 – 0.5Marginal — use with caution
< 0Unusable — controls not separated
Z' is only meaningful when comparing a positive control group to a negative control group. It is not a measure of hit effect.

S_pooled (Pooled Standard Deviation)

S_pooled_mean = √[((n_ctrl−1)·SD_ctrl² + (n_exp−1)·SD_exp²) / (n_ctrl+n_exp−2)]
S_pooled_median = √[((n_ctrl−1)·MAD_ctrl² + (n_exp−1)·MAD_exp²) / (n_ctrl+n_exp−2)]Robust

A degrees-of-freedom weighted average of the two groups' spread. Used as the noise estimate in Student's t-test and D-ratio. The median version uses MAD (×1.4826) in place of SD — because MAD is already scaled to be on the same unit as SD for normal data, the pooling formula is directly applicable. This gives a pooled noise estimate that is robust to outliers in either group.

D-ratio

D-ratio_mean = (z₀.₈ + z₀.₉₇₅) × S_pooled_mean × √(1/n_ctrl + 1/n_exp) / |mean_exp − mean_ctrl|
D-ratio_median = (z₀.₈ + z₀.₉₇₅) × S_pooled_median × √(1/n_ctrl + 1/n_exp) / |median_exp − median_ctrl|Robust

Where z₀.₈ + z₀.₉₇₅ = 0.8416 + 1.96 ≈ 2.80 (the constants for 80% power at α = 0.05, two-sided). The median version substitutes S_pooled_median (computed from MAD ×1.4826) and the median signal window — making it robust to outlier wells distorting the apparent PC–NC separation.

The numerator is the Minimum Detectable Difference (MDD) — the smallest signal difference that a two-sample t-test with the given n would reliably detect (power ≥ 80%, α = 0.05). The denominator is the actual observed PC–NC signal window.

D-ratio = MDD / Signal window. In other words: what fraction of the full PC–NC window does a compound need to reach before it can be statistically detected?

A D-ratio of 0.3 means any compound that reaches ≥ 30% of the PC effect will be detectable with >80% power. This makes 30% of PC a defensible hit threshold. The lower the D-ratio, the more sensitive the assay.

D-ratioAssay qualityPractical hit threshold
< 0.1Excellent — very wide signal windowCan set hit at 10–20% of PC effect
< 0.3Good — HTS standardSet hit at ~30% of PC effect
0.3 – 1.0Marginal — use with cautionRaise hit threshold accordingly
> 1.0Insufficient signal-to-noiseNot suitable for screening
D-ratio evaluates the quality of the assay window between a chosen PC and NC — it does not determine whether a compound is biologically meaningful. Always pair with Z' factor for a full QC picture.

Hypothesis Testing

Welch's t-test

t = |mean_exp − mean_ctrl| / √(SD_exp²/n_exp + SD_ctrl²/n_ctrl)

A modified Student's t-test that does not assume equal variance between groups (more appropriate than Student's t-test for most biological data). The Welch-Satterthwaite approximation is used for degrees of freedom:

df = (v_a + v_b)² / (v_a²/(n_a−1) + v_b²/(n_b−1)) where v = SD²/n

The resulting p-value is the probability of observing a difference this large (or larger) by chance alone, assuming the null hypothesis is true (no real difference). A small p-value (e.g. <0.05) is evidence against the null.

2-way ANOVA

Tests whether the mean response differs across groups (Treatment effect), across doses (Dose effect), and whether groups respond differently to dose (Interaction effect). This tool uses Type III sum of squares (effect coding), which gives equal weight to all groups regardless of sample size — consistent with Prism's default.

Multiple Testing Correction

When you perform many t-tests simultaneously, the probability of at least one false positive increases. For example, 30 tests at α=0.05 gives a ~78% chance of at least one spurious significant result by chance alone.

None
Raw p-values, no adjustment. Each test is evaluated at the nominal α level (e.g. 0.05).
Acceptable for 1–3 comparisons where you have a strong a priori hypothesis. Not appropriate for 10+ comparisons.
Holm (FWER — Family-Wise Error Rate)
A step-down procedure. Sort p-values in ascending order. Multiply the smallest p by m (total tests), the 2nd smallest by m−1, etc. Take the running maximum to ensure monotonicity. Controls the probability that any result in the family is a false positive at ≤ α.
Use when each significant star must be trustworthy. Confirmatory experiments, e.g. 30 siRNAs vs NTC where you need confidence in every hit before follow-up.
BH (FDR — Benjamini-Hochberg False Discovery Rate)
A step-up procedure. Sort p-values in ascending order. Adjusted p = p × m / rank. Controls the expected proportion of false positives among significant results at ≤ α. More lenient than Holm — more power, but allows some false positives.
Use for exploratory / screening experiments where you can validate hits in a follow-up assay. Appropriate for proteomics, genomics, or high-throughput compound screening (hundreds to thousands of tests).
MethodControlsPowerSuitable for
NonePer-comparison αHighest1–3 comparisons, pre-specified hypotheses
Holm (FWER)Any false positiveModerateConfirmatory, 5–50 comparisons

Outlier Removal

Outlier removal is applied per-group before computing statistics. Removed values are counted in the n_outlier column.

Grubbs test
Tests whether the most extreme value in a group is a statistical outlier. Assumes normality. Parameter α is the significance level for the test (default 0.05).
Best for small n (3–6 replicates) where other methods lack power. Standard choice for plate-based assays.
IQR (Interquartile Range)
Marks values outside Q1 − k·IQR to Q3 + k·IQR as outliers. Default k = 1.5 (standard "box" of a box plot). Does not assume normality.
Good for non-normal distributions or larger n (>6). More permissive than Grubbs at the default threshold. Increase k to 3 for a "far outlier" criterion.
ROUT
Robust outlier detection using a nonlinear regression-based approach (Prism's method). Parameter Q is the maximum acceptable FDR for outlier identification (default 1%).
Use as an alternative to Grubbs when normality cannot be assumed. Matches Prism's built-in ROUT outlier removal exactly.

Curve Fitting & EC50

4-Parameter Logistic (4PL)

y = d + (a − d) / (1 + (x/c)^b)
ParameterMeaningBiological interpretation
aTop asymptoteBaseline response (zero drug)
bHill coefficient / slopeSteepness of the curve. b>0 = activator, b<0 = inhibitor
cEC50 (inflection point)Concentration producing 50% of maximum effect
dBottom asymptoteMaximum response at saturating concentration

EC50

The half-maximal effective concentration — the dose that produces 50% of the maximum effect defined by the 4PL fit. EC50 is derived from parameter c and is reported in the same units as the dose column.

EC50 values are log-normally distributed. Always compare log(EC50) values when doing statistics across groups.
EC50 fitting requires at least 4 dose points spanning the response range (ideally top and bottom plateaus reached). Fitting may fail or give unreliable results with sparse data.
